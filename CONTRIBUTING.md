# Вклад (Contributing) в AIFixed

# Руководство по внесению вклада в AIFixed

Спасибо за интерес к проекту **AIFixed**. Этот документ описывает, как правильно участвовать в разработке, предлагать улучшения и отправлять пулреквесты.

---

## 1. Общие принципы

- Все изменения должны быть обоснованными.
- Код должен быть чистым, консистентным и соответствовать стилю проекта.
- Перед отправкой PR протестируйте изменения локально.
- Коммиты должны быть информативными.

---

## 2. Как начать

### Создайте форк

```bash
git clone https://github.com/<your-username>/AIFixed.git
cd AIFixed
git remote add upstream https://github.com/aklyue/AIFixed.git
```

### Создайте ветку

```bash
git checkout -b feature/<название>
```

Форматы веток:

- `feature/<имя>`
- `fix/<имя>`
- `refactor/<имя>`
- `docs/<имя>`
- `chore/<имя>`

---

## 3. Локальная разработка

### Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm start
```

### Docker

```bash
docker-compose up --build
```

---

## 4. Правила написания кода

### Backend (FastAPI + Python)

- Следовать PEP8.
- Форматировать код `black` и `isort`.
- Поддерживать типизацию.
- Новые эндпоинты должны иметь Pydantic-схемы.
- Логика должна быть вынесена в `services/`.

### Frontend (React + TypeScript)

- Типизация обязательна.
- Не использовать `any`.
- Соблюдать архитектуру FSD: `shared/`, `entities/`, `features/`, `pages/`.
- Компоненты должны быть чистыми и переиспользуемыми.

---

## 5. Тестирование

Перед созданием PR:

- Протестируйте основной функционал.
- Проверьте генерацию презентации.
- Проверьте работу редактора и API.
- Убедитесь, что Docker-сборка проходит без ошибок.

---

## 6. Pull Request

PR должен содержать:

- Описание изменений.
- Список связанных issues (если есть).
- Пояснение мотивации.
- Скриншоты (если изменения фронтенда).

---

## 7. Стиль коммитов

Используем соглашение:

```
feature: добавлена новая возможность
fix: исправлена ошибка
refactor: рефакторинг
docs: документация
chore: техническое изменение
```

---

## 8. Кодекс поведения

Участники должны уважать друг друга и сохранять профессионализм.

---

## Лицензия

Внося вклад в этот проект, вы соглашаетесь с тем, что ваши изменения будут распространяться под лицензией [MIT](https://github.com/aklyue/AIFixed/blob/main/LICENSE), используемой в проекте.

Все ваши улучшения, исправления ошибок, новые функции и правки документации становятся частью проекта и также будут доступны под лицензией MIT.

---

Спасибо за ваш вклад в развитие AIFixed! Ваши усилия помогают улучшать систему, исправлять ошибки и добавлять полезные функции для всех пользователей.

Мы ценим ваше время, идеи и экспертизу. Каждый вклад, даже небольшой, имеет значение.
